# 接入 figma_rest MCP Server 流程

## 全局安装

1. 创建 Figma Personal Access Token

在 Figma 主页进入：

```text
Settings -> Security -> Personal access tokens -> Generate new token
```

权限建议：

- `file_content:read`：必选，用于读取设计稿文件和节点内容。
- `file_comments:read`：可选，只有需要 AI 读取评论、备注、交互说明时再勾选。

2. 安装 MCP 包

```bash
npm i -g figma-rest-mcp-server
```

也可以使用 pnpm：

```bash
pnpm add -g figma-rest-mcp-server
```

3. 写入 Figma PAT

```bash
figma-rest-mcp auth
```

命令会在用户主目录生成本机凭据文件：

```text
~/.figma-rest-mcp-server/auth.json
```

`auth.json` 是本机明文凭据文件，不能提交、不能共享。每个人都应使用自己的 Figma Personal Access Token。

4. 查看运行路径

```bash
figma-rest-mcp paths
```

默认路径：

```text
~/.figma-rest-mcp-server/auth.json
~/.figma-rest-mcp-server/.figma-cache
```

5. 需要重新拉取设计稿时，可以清空本地快照缓存

```bash
figma-rest-mcp cache:clear
```

该命令只会删除并重建：

```text
~/.figma-rest-mcp-server/.figma-cache
```

不会删除 `auth.json`。

6. 注册 MCP 到 Codex

```bash
codex mcp add figma_rest -- figma-rest-mcp serve
```

7. 重启 Codex App 或开启新会话

在新会话中即可调用：

```text
使用 figma_rest 的 figma_fetch_snapshot 读取这个 Figma 链接：...
```

## 重新注册

查看已有配置：

```bash
codex mcp get figma_rest
```

删除已有配置：

```bash
codex mcp remove figma_rest
```

重新注册：

```bash
codex mcp add figma_rest -- figma-rest-mcp serve
```

## 源码开发

从源码运行时：

```bash
cd <figma-rest-mcp-server项目目录>
pnpm install
pnpm build
pnpm cli auth
pnpm cli paths
pnpm cli cache:clear
pnpm cli serve
```

源码开发期注册到 Codex：

```bash
codex mcp add figma_rest -- node <figma-rest-mcp-server项目目录>/dist/cli.js serve
```

认证文件和缓存目录仍使用统一的用户目录：

```text
~/.figma-rest-mcp-server/auth.json
~/.figma-rest-mcp-server/.figma-cache
```

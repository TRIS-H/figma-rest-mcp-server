原理是：这个 MCP 本身不“理解 Figma 文件”，它是一个中间层，把 Codex 的 MCP 调用转成 Figma REST API 请求，再把 Figma 返回的 JSON 缓存和按需读取。

流程是：

```text
Codex
  -> figma_rest MCP 工具
  -> 本地 figma-rest-mcp-server
  -> Figma REST API
  -> Figma 文件 JSON / 节点 JSON / 图片导出 URL
  -> 本地缓存
  -> Codex 读取节点数据用于开发
```

核心数据来源是 Figma 官方 REST API：

```text
GET /v1/files/:file_key
GET /v1/files/:file_key/nodes?ids=...
GET /v1/images/:file_key?ids=...
```

比如你给的链接：

```text
https://www.figma.com/design/Lxci9PlOEhEnJ7D0EJLm5e/...?node-id=1974-7822
```

项目会解析出：

```text
fileKey = Lxci9PlOEhEnJ7D0EJLm5e
nodeId = 1974:7822
```

然后请求 Figma：

```text
/v1/files/Lxci9PlOEhEnJ7D0EJLm5e
/v1/files/Lxci9PlOEhEnJ7D0EJLm5e/nodes?ids=1974:7822
/v1/images/Lxci9PlOEhEnJ7D0EJLm5e?ids=1974:7822&format=png
```

Figma 返回的节点 JSON 里本来就包含样式开发需要的信息，例如：

- `absoluteBoundingBox`：位置、宽高
- `constraints`：约束
- `layoutMode`：Auto Layout 方向
- `primaryAxisSizingMode` / `counterAxisSizingMode`
- `itemSpacing`：间距
- `paddingLeft` / `paddingRight` / `paddingTop` / `paddingBottom`
- `fills`：背景色、图片填充、渐变
- `strokes`：描边
- `cornerRadius`：圆角
- `effects`：阴影、模糊
- `style`：文字字体、字号、行高、字重
- `characters`：文本内容
- `children`：子节点层级
- `componentId` / `componentProperties`：组件实例信息

项目做的事情主要是：

1. 用 PAT 鉴权访问 Figma API  
   PAT 从：

```text
~/.figma-rest-mcp-server/auth.json
```

读取。

2. 拉取完整文件 JSON  
   代码在 [src/figmaClient.ts](/src/figmaClient.ts)。

3. 生成节点索引  
   代码在 [src/nodeIndex.ts](/src/nodeIndex.ts)。

4. 缓存快照  
   写到：

```text
~/.figma-rest-mcp-server/.figma-cache
```

代码在 [src/snapshotStore.ts](/src/snapshotStore.ts)。

5. 提供 MCP 工具给 Codex 用  
   代码在 [src/toolHandlers.ts](/src/toolHandlers.ts)。

所以它不是截图识别，也不是爬网页，而是读取 Figma 官方接口返回的结构化设计数据。限制是：REST API 能拿到大量布局/样式信息，但不等同 Figma 官方 Dev Mode MCP，原型交互、Dev Mode 语义增强不保证完整。
